# observations folder

this directory saves cleaned audio files generated during tests so that we can listen to them later and share them in meetings.

## expected output files generated during evaluation:

- `enhanced_noisereduce_ami.wav` -- ami ihm sample enhanced with noisereduce (stoi: 0.863, si-sdr: 4.846 db, dnsmos: 2.2572, 16000hz)
- `enhanced_deepfilter_ami.wav` -- ami ihm sample enhanced with deepfilternet3 (stoi: 0.9988, si-sdr: 36.638 db, dnsmos: 3.0126, aligned 16000hz)
- `outputs/diarization_sample_input.json` -- complete set of "who spoke when" timestamps generated by pyannote in phase 2.
- `outputs/enhanced_sample_input.wav` -- the phase 1 deepfilter output that is fed directly into phase 2 diarization.

## inspection instructions
to test the enhancement suite independently:
```bash
python -m pytest tests/ -v -s
```

to run the full end-to-end pipeline (enhancement + diarization):
```bash
python run_pipeline.py --input screening_notebooks/sample_input_and_output_files/sample_input.wav --output-dir observations/outputs --token YOUR_HF_TOKEN
```
